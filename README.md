# GeoJSON Zones

Десктоп-приложение для Windows (с заделом под macOS), которое хранит и
систематизирует **зоны** (полигоны GeoJSON), сгруппированные по **городам**.

Зоны — это области, очерченные в агломерации города. Они используются сервисом
расчёта стоимости перевозки и проверки входимости адреса клиента в зону.

## Что умеет (целевой функционал)

- 📁 **Справочник городов** — создание, переименование, удаление (с CRUD).
- 🗺️ **Зоны** — хранение исходного GeoJSON целиком, привязка к городу.
- ⬆️ **Drag&drop загрузка** .geojson файлов (поодиночке и массово).
- 🏷️ **Логика «сначала зона → потом город»** — загруженная зона попадает в раздел
  «не выбран город» (подсвечен), затем ей назначают город.
- 🌳 **Иерархический журнал** «город → зоны» с датами создания и изменения.
- ⬇️ По каждой зоне — **скачать GeoJSON** и **скачать XLSX** (широта/долгота).
- 🗺️ **Отображение полигона на карте** (Leaflet + OpenStreetMap).
- 📝 **Журнал действий** в отдельной вкладке.
- ✏️ **Переименование** городов и зон в программе **не меняет** сами файлы.

## Технологии

- **Electron** + **electron-builder** (установщик `.exe`, NSIS)
- **SQLite** (`better-sqlite3`) — локальное хранилище
- **SheetJS (xlsx)** — генерация Excel
- **Leaflet** — карта

## Разработка

```bash
npm install        # установить зависимости
npm start          # запустить приложение в режиме разработки

# Чистые node-тесты (без Electron):
npm run test:geojson  # разбор GeoJSON и генерация XLSX
npm run test:pip      # входимость точки в полигон (ray-casting)
npm run test:geocode  # разбор ответа геокодера Nominatim
npm run test:polygon  # построение полигона из вершин
npm run test:sync     # логика синхронизации (хэш, вердикт актуальности)
npm run test:folders  # экспорт в папки
npm run test:bugreport

# Electron-смоуки (изолированная БД, env GZ_*):
npm run test:db    # слой данных
npm run test:ui    # обойти все вкладки
npm run test:func  # путь renderer→IPC→БД
npm run test:badge # счётчик зон без города
npm run test:map   # вкладка «Карта»
npm run test:addr  # вкладка «Проверка адреса»
npm run test:draw  # вкладка «Создание полигона»
npm run test:sync-ui  # вкладка «Обновление данных»
```

## Сборка Windows-установщика (.exe)

`better-sqlite3` — нативный модуль, и его **нельзя кросс-компилировать под Windows
с macOS** (получится mac-бинарник, приложение упадёт на Windows). Поэтому
установщик собирается на настоящей Windows:

- **Рекомендуется — через GitHub Actions** (Windows-машина не нужна):
  вкладка **Actions → Build Windows installer → Run workflow**. Готовый `.exe`
  появится в артефактах сборки. При пуше тега `vX.Y.Z` — прикрепляется к Release.
- **Локально на Windows:** `npm install && npm run dist:win` → `dist/…Setup.exe`.

На macOS `npm run dist:win` формально отрабатывает, но получившийся `.exe`
нерабочий (внутри mac-версия native-модуля) — использовать нельзя.
